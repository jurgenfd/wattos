This package contains Java code specific to the molecular objects in a soup. (WHAT IF jargon)

For the specific entities like atom, residue, molecule, etc. look in the Atom class for the most complete documentation.

The Atoms class is the basic object which is optimized to deal with many atoms. The classes above it (like Residue/Molecule/Ensemble/Gumbo) organize the atoms in a more object-oriented fashion. The classes like {@link Wattos.Soup.Atom} use primitive data types and non-redundant strings as made possible by the {@link Wattos.Database} package in order to prevent the generation of many objects like Strings which is inefficient in Java beyond the order of a thousand or so.

Recap of the order with mmCIF and NMR-STAR equivalents

-@-1- Atom Atom
-:-2- Residue Chemical Component
-#-3- Molecule Entity
-~-4- Model Assembly
-^-5- Entry Same
---6- DBMS Same

Everything is in one soup (called Gumbo in Wattos).

In the future a parser will be implemented that can be used to do things like: "make an atom taste good" (in MOLMOL jargon): 

SetProperty @Atom:Residue#Molecule^Ensemble 'jummie'

Estimated maximum number of atoms in the gumbo 

-@-1- Atom
-:-2- Residue has 20 atoms (including pseudo atoms) 10,000,000 atoms
-#-3- Molecule has 100 residues (ions will be a residue when bound) 500,000 residues
-~-4- Model has 10 molecules (e.g. multimer) 5,000 molecules
-^-5- Entry has 50 500 models
---6- DBMS has 10 entries 10 entry

Total 10,000,000 atoms for biggest and most complex case above. Reading/writing/etc. of such quantities is a matter of seconds in Wattos.
